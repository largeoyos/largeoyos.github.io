# 模块 5：信源编码（数据压缩）

到现在为止，我们一直在抽象世界里讨论"信息"。这一节我们终于把它**落地到比特和字节**——回答一个具体的工程问题：

> 给我一段数据，我能把它压缩到多小？极限在哪？

答案就是熵。这是香农 1948 年那篇论文的第一个核心定理，叫做**信源编码定理**（Source Coding Theorem）。

---

### 一、问题的设定

#### 信源是什么

一个**信源**（Source）就是一个产生随机符号的东西。形式上，就是一个随机变量 XX，每次取值来自某个字母表 X={x1,x2,…,xn}X={x1​,x2​,…,xn​}。

举几个例子：

- 英文文本：字母表是 26 个字母 + 空格 + 标点
- 灰度图像：字母表是 0-255 共 256 个灰度值
- DNA 序列：字母表是 {A,T,C,G}{A,T,C,G}
- 抛硬币：字母表是 {0,1}{0,1}

每个符号 xixi​ 出现的概率是 p(xi)p(xi​)。**这些概率不一定均匀**——这一点是数据可压缩的根源。

#### 编码是什么

**编码**就是把信源的每个符号 xixi​ 映射成一串 0/1 比特，称为**码字**（codeword）。

举例：英文字母编码

|字母|码字|
|---|---|
|A|`000`|
|B|`001`|
|...|...|
|Z|`11010`|

编码后的比特串可以传输、存储、解码还原。

#### 我们关心什么

设符号 xixi​ 的码字长度为 ℓiℓi​ 比特。**平均码长**：

L=∑ip(xi)⋅ℓi=E[ℓ]L=i∑​p(xi​)⋅ℓi​=E[ℓ]

**核心问题**：怎么设计编码，让 LL 尽量小？

---

### 二、关键直觉：常见的给短码字

观察一个事实：英文里 E 出现的频率约 12%，而 Z 只有 0.07%。如果我给 E 和 Z 都用相同长度的码字，那是浪费——**应该让常出现的符号用短码字，罕见的符号用长码字**。

这就是著名的"摩尔斯电码"思想：E 是一个点（`.`），Z 是 `--..`（4 个符号）。

数学上，这正好对应模块 1 的洞察：**信息量 −log⁡p(x)−logp(x) 越大的符号越罕见，应该用越长的码字**。

理想情况下，符号 xixi​ 的码长应该接近：

ℓi≈−log⁡2p(xi)ℓi​≈−log2​p(xi​)

如果真能做到这样，平均码长就是：

L=∑ip(xi)⋅(−log⁡2p(xi))=H(X)L=i∑​p(xi​)⋅(−log2​p(xi​))=H(X)

**平均码长 = 熵**。这就是模块 2 我说"熵是最优编码长度下限"的真正含义。

但现实里 −log⁡2p(xi)−log2​p(xi​) 一般不是整数，码长必须是整数，所以实际编码总会比熵稍长。下面我们看具体怎么做。

---

### 三、好的编码要满足什么条件

#### 条件 1：可译码

如果两个不同符号被编成相同的码字，解码就乱套了。所以**不同符号要有不同码字**——这叫**唯一可译**。

#### 条件 2：即时可译（前缀码）

更强的要求：码字流不需要"等下文"就能解码。也就是说，**任何码字都不能是另一个码字的前缀**。

举例对比：

||不好的编码|好的编码|
|---|---|---|
|A|`0`|`0`|
|B|`01`|`10`|
|C|`011`|`110`|
|D|`0111`|`111`|

不好的编码里：A 是 B 的前缀，B 是 C 的前缀。当你接收 `0111` 时，你不知道这是"A 接 BBB"还是"D"——必须看完才能反推，无法实时解码。

满足"无前缀"性质的编码叫**前缀码**（prefix code）或**即时码**。我们要找的就是好的前缀码。

#### Kraft 不等式（前缀码存在的条件）

定理：码长为 ℓ1,ℓ2,…,ℓnℓ1​,ℓ2​,…,ℓn​ 的二元前缀码存在，**当且仅当**：

∑i=1n2−ℓi≤1i=1∑n​2−ℓi​≤1

这叫 **Kraft 不等式**。

直观理解：你可以把所有可能的二进制串想象成一棵二叉树。每用掉一个长度为 ℓℓ 的码字，就"占据"了一个节点，并禁用了它的所有子孙——这相当于占了 2−ℓ2−ℓ 比例的"码字空间"。所有码字占用的空间不能超过 1。

证明思路（不严格）：把所有长度为 L=max⁡ℓiL=maxℓi​ 的二进制串想成一个 2L2L 大小的池子。一个长度为 ℓiℓi​ 的码字"覆盖"了 2L−ℓi2L−ℓi​ 个长串（它的所有延伸）。前缀码不允许覆盖重叠，所以总覆盖不超过 2L2L：

∑i2L−ℓi≤2L  ⟹  ∑i2−ℓi≤1i∑​2L−ℓi​≤2L⟹i∑​2−ℓi​≤1

### 四、信源编码定理（香农第一定理）

我们现在可以把问题精确化：

> **在所有前缀码中，最小的平均码长 L∗L∗ 是多少？**

#### 下界

定理（信源编码定理 · 下界）：任何前缀码满足

L≥H(X)L≥H(X)

**平均码长不可能小于熵。** 这是一条铁律——无论你多聪明，多巧妙地设计编码，都不能突破熵这道墙。

#### 证明

利用 Kraft 不等式 ∑i2−ℓi≤1∑i​2−ℓi​≤1。我们要证 L=∑ipiℓi≥H(X)L=∑i​pi​ℓi​≥H(X)。

考虑差值：

L−H(X)=∑ipiℓi+∑ipilog⁡2pi=∑ipilog⁡22ℓi+∑ipilog⁡2pi=∑ipilog⁡2pi2−ℓiL−H(X)​=i∑​pi​ℓi​+i∑​pi​log2​pi​=i∑​pi​log2​2ℓi​+i∑​pi​log2​pi​=i∑​pi​log2​2−ℓi​pi​​​

构造一个分布 qi=2−ℓicqi​=c2−ℓi​​，其中 c=∑i2−ℓi≤1c=∑i​2−ℓi​≤1（这样 ∑qi=1∑qi​=1）。则：

2−ℓi=c⋅qi2−ℓi​=c⋅qi​

代入：

L−H(X)=∑ipilog⁡2pic⋅qi=∑ipilog⁡2piqi−log⁡2c=DKL(P ∥ Q)−log⁡2cL−H(X)​=i∑​pi​log2​c⋅qi​pi​​=i∑​pi​log2​qi​pi​​−log2​c=DKL​(P∥Q)−log2​c​

由 Gibbs 不等式 DKL(P ∥ Q)≥0DKL​(P∥Q)≥0，由 c≤1c≤1 得 −log⁡2c≥0−log2​c≥0，所以：

L−H(X)≥0  ⟹  L≥H(X)■L−H(X)≥0⟹L≥H(X)■

**这就是模块 4 学的 KL 散度的第一次大显身手**——证明信息论最重要的下界定理。

#### 上界

定理（信源编码定理 · 上界）：存在前缀码满足

L<H(X)+1L<H(X)+1

**证明思路**：取 ℓi=⌈−log⁡2pi⌉ℓi​=⌈−log2​pi​⌉（即取整到不小于的最近整数）。可以验证它满足 Kraft 不等式（前缀码存在），平均码长满足上界。

#### 合在一起

 H(X)≤L∗<H(X)+1 H(X)≤L∗<H(X)+1​

**最优前缀码的平均码长在 H(X)H(X) 到 H(X)+1H(X)+1 之间。**

那个 +1+1 是因为码长必须是整数造成的"取整开销"。如果我们一次编码很多个符号（块编码），把开销摊薄到每个符号上，可以无限接近 H(X)H(X)：

Ln∗n→H(X) as n→∞nLn∗​​→H(X) as n→∞

这就是**香农第一定理的完整形式**：**熵是数据压缩的不可逾越的极限，但可以无限接近**。

### 五、Huffman 编码——一种实用的最优编码

理论说"存在 L<H+1L<H+1 的前缀码"，但具体怎么造？

**Huffman 编码**（1952，David Huffman 当时还是 MIT 研究生）给出了构造方法，并且证明了它是**最优前缀码**——没有任何前缀码能比 Huffman 平均码长更短。

#### 算法

1. 把所有符号按概率从小到大排列。
2. 取**两个最小概率**的符号，合并成一个新节点，概率 = 两者之和。
3. 把这个新节点放回队列，重复第 2 步。
4. 直到只剩一个节点（根）。
5. 从根出发，左分支标 `0`，右分支标 `1`，叶子节点的路径就是该符号的码字。

#### 例子

设信源 XX 取 5 个符号，概率分别为：

|符号|概率|
|---|---|
|A|0.4|
|B|0.2|
|C|0.2|
|D|0.1|
|E|0.1|

**Step 1**：合并最小的 D(0.1) 和 E(0.1) → DE(0.2)

剩下：A(0.4), B(0.2), C(0.2), DE(0.2)

**Step 2**：合并最小的两个，比如 C(0.2) 和 DE(0.2) → CDE(0.4)

剩下：A(0.4), B(0.2), CDE(0.4)

**Step 3**：合并 B(0.2) 和 CDE(0.4) → BCDE(0.6)

剩下：A(0.4), BCDE(0.6)

**Step 4**：合并 → 根(1.0)

构造的二叉树：

```
            根 (1.0)
           /        \
         0/          \1
         /            \
       A(0.4)       BCDE(0.6)
                    /        \
                   0/          \1
                   /            \
                B(0.2)         CDE(0.4)
                                /     \
                              0/       \1
                              /         \
                           C(0.2)      DE(0.2)
                                       /     \
                                     0/       \1
                                     /         \
                                  D(0.1)     E(0.1)
```

读出码字：

|符号|概率|码字|长度|
|---|---|---|---|
|A|0.4|`0`|1|
|B|0.2|`10`|2|
|C|0.2|`110`|3|
|D|0.1|`1110`|4|
|E|0.1|`1111`|4|

#### 验证

**平均码长**：

L=0.4⋅1+0.2⋅2+0.2⋅3+0.1⋅4+0.1⋅4=0.4+0.4+0.6+0.4+0.4=2.2L=0.4⋅1+0.2⋅2+0.2⋅3+0.1⋅4+0.1⋅4=0.4+0.4+0.6+0.4+0.4=2.2

**信源熵**：

H(X)=−0.4log⁡20.4−0.2log⁡20.2−0.2log⁡20.2−0.1log⁡20.1−0.1log⁡20.1≈0.529+0.464+0.464+0.332+0.332≈2.122 比特H(X)​=−0.4log2​0.4−0.2log2​0.2−0.2log2​0.2−0.1log2​0.1−0.1log2​0.1≈0.529+0.464+0.464+0.332+0.332≈2.122 比特​

**比较**：

- 熵 H(X)≈2.12H(X)≈2.12
- Huffman 平均码长 L=2.2L=2.2
- 用普通定长码（每个符号 3 比特）：L定长=3L定长​=3

Huffman 比定长码省了约 27%，且**已经非常接近熵的极限 2.12**。

#### Huffman 的最优性

定理：在所有逐符号前缀码中，Huffman 编码使平均码长最小。

证明依赖几个引理（最优码中概率最小的两个符号有相同的最长码长且只在最后一位不同），整体不复杂但有点繁琐，这里跳过。**记住结论：Huffman 是逐符号编码的最优方案**。

#### Huffman 的局限

Huffman 的码长仍受"整数"限制，所以达不到熵。例：信源 {A:0.99,B:0.01}{A:0.99,B:0.01}，熵约 0.080.08 比特，但 Huffman 必须给每个符号至少 1 比特，平均码长 = 1，**比熵高 12 倍**！

解决方案有两条路：

1. **块编码**：把多个符号打包再编码，分散整数损失。
2. **算术编码 / 区间编码**（更现代）：直接产生分数比特数的码长，可以无限接近熵。这是 ZIP、JPEG 等现代压缩算法的核心。

### 六、回到现实——这些理论藏在哪里

电子信息工程的同学会在很多地方碰到这些算法：

|场景|用了什么|为什么|
|---|---|---|
|ZIP / GZIP / DEFLATE|Huffman + LZ77|文件压缩|
|JPEG|Huffman 编码 DCT 系数|图像压缩|
|MP3 / AAC|Huffman 编码频谱系数|音频压缩|
|HEVC / H.265|算术编码（CABAC）|视频压缩|
|文件传输协议|前缀码|流式解码|

每一个都是香农第一定理的工程实现。

更深刻的是：**所有这些压缩格式的极限性能，都被信源的熵卡住**。无论 ZIP 算法再怎么进化，压一段英文文本，最终极限就是英文的熵（约 1.5 比特/字符）。这是物理定律级别的限制。

### 七、一个关键认识：压缩的本质

我用一句话总结这一节：

> **数据可以被压缩，是因为它的分布不均匀（熵小于 log⁡nlogn）。完全均匀分布的数据是不可压缩的——它每个比特都是"满信息"的。**

这是为什么：

- 已经压缩过的文件再压一次几乎没效果（接近均匀，熵接近最大）。
- 加密后的数据看起来像随机噪声，无法压缩（好的加密让分布近似均匀）。
- "压缩比"反映了原始数据的冗余程度。

熵衡量的就是**数据中的"非冗余信息量"**——压不掉的那部分。

### 八、本模块小结

**核心定理（香农第一定理）**：

H(X)≤L∗<H(X)+1H(X)≤L∗<H(X)+1

最优前缀码的平均码长被熵卡住——**熵是数据压缩的极限**。

**两个证明工具**：

- Kraft 不等式：前缀码存在的充要条件 ∑i2−ℓi≤1∑i​2−ℓi​≤1
- Gibbs 不等式：用来证下界 L≥H(X)L≥H(X)

**实用算法**：

- Huffman 编码：逐符号最优，但受整数限制
- 算术编码：突破整数限制，可逼近熵

**核心认知**：

- 数据压缩的极限是熵 H(X)H(X)
- 分布越不均匀，熵越小，压缩潜力越大
- 这是物理定律级别的限制

### 接下来

模块 5 讲的是**没有噪声**的情况——你输入什么、输出就是什么，只是要让传输更省。

但现实通信里，信道**有噪声**：你发 0 可能收到 1。这时候问题完全变了：

- 怎么定义"有噪声信道的传输能力"？
- 加冗余（编码）能不能克服噪声？
- 在多大速率下还能保证"几乎无错"传输？

这些问题的答案就是**信道容量**和**香农第二定理**——信息论的最高峰。
