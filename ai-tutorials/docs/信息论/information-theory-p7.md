# 模块 6：信道与信道容量

模块 5 处理的是"完美信道"：你发 0，对方就收到 0。我们在压缩上做文章。

但**现实里没有完美信道**。你打电话有杂音，Wi-Fi 信号会衰减，硬盘会出现位翻转，深空探测器传回的图像里有噪声——通信的真实图景是：

```
   X  ──→  [有噪信道]  ──→  Y
  发送              接收
```

发送 XX，但接收的 YY **不一定等于 XX**。这就引出香农工作的另一半，也是信息论的最高峰：

> **在有噪声的信道里，能不能可靠通信？最快能多快？**

这一节先建立信道模型，定义"信道容量"，并算几个具体例子。下一节（模块 7）我们会看到震撼世界的香农第二定理。

---

### 一、信道的数学模型

#### 信道是什么

一个**离散无记忆信道**（Discrete Memoryless Channel, DMC）由三部分组成：

- **输入字母表** XX：发送端可以发的符号
- **输出字母表** YY：接收端可能收到的符号
- **转移概率** p(y∣x)p(y∣x)：发送 xx 时收到 yy 的概率

"无记忆"的意思是：每次传输独立——这一次的噪声不影响下一次。

整个信道完全由**转移概率矩阵** P(Y∣X)P(Y∣X) 描述：

P=(p(y1∣x1)p(y2∣x1)⋯p(y1∣x2)p(y2∣x2)⋯⋮⋮⋱)P=​p(y1​∣x1​)p(y1​∣x2​)⋮​p(y2​∣x1​)p(y2​∣x2​)⋮​⋯⋯⋱​​

每一**行**是一个条件分布（给定输入 xx，输出 yy 的分布），每行求和为 1。

#### 输入分布是另一回事

注意：**输入分布 p(x)p(x) 不是信道本身的属性，是你（发送方）可以选择的**。

信道决定 p(y∣x)p(y∣x)（噪声特性，物理属性），你决定 p(x)p(x)（发什么、按什么频率发）。

接收端的分布由两者一起决定：

p(y)=∑xp(x)⋅p(y∣x)p(y)=x∑​p(x)⋅p(y∣x)

### 二、三个标准信道例子

#### 例 1：无噪信道（Noiseless Channel）

X=Y={0,1}X=Y={0,1}，转移矩阵：

P=(1001)P=(10​01​)

发啥收啥，零噪声。这是模块 5 的设定。

#### 例 2：二元对称信道（Binary Symmetric Channel, BSC）

最经典的有噪信道。X=Y={0,1}X=Y={0,1}，每个比特有 pp 的概率被翻转：

P=(1−ppp1−p)P=(1−pp​p1−p​)

```
       1-p
   0 ──────→ 0
    \      /
     \  p /
      \  /
       \/
       /\
      /  \
     /  p \
    /      \
   1 ──────→ 1
       1-p
```

参数 pp 叫**信道翻转概率**，0≤p≤1/20≤p≤1/2。p=0p=0 是无噪信道，p=1/2p=1/2 是完全随机（最坏）。

物理对应：噪声大致对称的硬盘、无线信道。

#### 例 3：二元擦除信道（Binary Erasure Channel, BEC）

X={0,1}X={0,1}，Y={0,1,e}Y={0,1,e}（多了一个"擦除"符号）。每个比特以概率 ϵϵ 被丢失（变成 ee），否则正确传输：

P=(1−ϵ0ϵ01−ϵϵ)P=(1−ϵ0​01−ϵ​ϵϵ​)

物理对应：网络丢包（你知道哪一个包丢了，但不知道内容）。

### 三、信道容量的定义

我们想问：**这个信道一次传输，最多能传多少信息？**

#### 一次传输能传多少信息？答案是 I(X;Y)I(X;Y)

回忆模块 3：

I(X;Y)=H(X)−H(X∣Y)I(X;Y)=H(X)−H(X∣Y)

直观读：**XX 原本携带 H(X)H(X) 比特信息，经过信道后，由于噪声，接收端 YY 中只能"看到" I(X;Y)I(X;Y) 比特关于 XX 的信息**——剩下的 H(X∣Y)H(X∣Y) 比特被噪声"吃掉"了。

所以：**一次信道使用所传输的信息量 = I(X;Y)I(X;Y)**。

#### 但 I(X;Y)I(X;Y) 同时取决于信道和输入分布

I(X;Y)I(X;Y) 由 p(x,y)=p(x)p(y∣x)p(x,y)=p(x)p(y∣x) 决定。其中：

- p(y∣x)p(y∣x) 由信道决定（你不能改）
- p(x)p(x) 由你决定（你能改）

那么我们当然要**选一个最好的 p(x)p(x)**，让 I(X;Y)I(X;Y) 最大。

#### 信道容量的定义

 C=max⁡p(x)I(X;Y) C=p(x)max​I(X;Y)​

**信道容量**是在所有可能的输入分布上取最大互信息。它**只是信道的属性**——一旦信道给定，CC 就唯一确定。

单位：比特/信道使用（bits per channel use）。

### 四、计算容量例 1：无噪信道

X=Y={0,1}X=Y={0,1}，Y=XY=X。

H(X∣Y)=0H(X∣Y)=0（已知 YY 完全确定 XX）。

I(X;Y)=H(X)−0=H(X)I(X;Y)=H(X)−0=H(X)

最大化 H(X)H(X) → 取均匀分布 p(0)=p(1)=1/2p(0)=p(1)=1/2 → H(X)=1H(X)=1。

C=1 比特/使用C=1 比特/使用

**直觉**：无噪信道一次传 1 比特。完全合理。

### 五、计算容量例 2：二元对称信道（BSC）

这是信息论里最经典的容量计算，仔细做一遍。

#### 列出 I(X;Y)I(X;Y)

I(X;Y)=H(Y)−H(Y∣X)I(X;Y)=H(Y)−H(Y∣X)

我们用这个分解（不用 H(X)−H(X∣Y)H(X)−H(X∣Y)）有特别的理由——下面会看到 H(Y∣X)H(Y∣X) 跟输入分布无关。

#### 算 H(Y∣X)H(Y∣X)

H(Y∣X)=∑xp(x)H(Y∣X=x)H(Y∣X)=x∑​p(x)H(Y∣X=x)

固定 X=0X=0：YY 是"以 1−p1−p 概率为 0、以 pp 概率为 1"的分布。它的熵是二元熵函数：

H(Y∣X=0)=H2(p):=−plog⁡2p−(1−p)log⁡2(1−p)H(Y∣X=0)=H2​(p):=−plog2​p−(1−p)log2​(1−p)

固定 X=1X=1：对称地，H(Y∣X=1)=H2(p)H(Y∣X=1)=H2​(p)。

所以：

H(Y∣X)=p(0)H2(p)+p(1)H2(p)=H2(p)H(Y∣X)=p(0)H2​(p)+p(1)H2​(p)=H2​(p)

**H(Y∣X)=H2(p)H(Y∣X)=H2​(p) 与输入分布无关**！它只取决于信道的噪声参数。

物理含义：每次传输，无论你发什么，**信道都会注入 H2(p)H2​(p) 比特的"噪声不确定性"**。这是噪声的"信息税"。

#### 最大化 H(Y)H(Y)

I(X;Y)=H(Y)−H2(p)I(X;Y)=H(Y)−H2​(p)

第二项是常数，要最大化 II 就要最大化 H(Y)H(Y)。

YY 是二元的，H(Y)≤1H(Y)≤1（模块 2 的最大熵定理），等号当 YY 均匀。

我们能让 YY 均匀吗？由 BSC 的对称性：如果 p(0)=p(1)=1/2p(0)=p(1)=1/2（输入均匀），那 p(Y=0)=p(Y=1)=1/2p(Y=0)=p(Y=1)=1/2（输出也均匀）。**可以**。

#### 容量

 CBSC=1−H2(p) CBSC​=1−H2​(p)​

#### 验证几个特殊值

|pp|H2(p)H2​(p)|CC|含义|
|---|---|---|---|
|0|0|1|无噪信道，满速|
|0.1|0.469|0.531|10% 错误率|
|0.3|0.881|0.119|30% 错误率，容量大降|
|**0.5**|**1**|**0**|**完全随机，无法通信**|
|0.9|0.469|0.531|等于 p=0.1p=0.1 !|
|1|0|1|必翻转，等价于无噪|

#### 三个值得品味的现象

**现象 1：p=0.5p=0.5 时 C=0C=0**

完全随机的信道一点信息都传不了。这跟直觉一致：收到 YY 后，YY 与 XX 完全无关，你无法推断 XX。

**现象 2：p=0p=0 和 p=1p=1 容量相同**

p=1p=1 意思是"必翻转"——你发 0 我必收 1，发 1 必收 0。但我**知道**它必翻转，把收到的取反就行了，等价于无噪信道。

启示：**噪声的"确定性"不是问题，"随机性"才是问题**。最坏的不是"必翻转"，而是"不知道翻不翻"。

**现象 3：CC 关于 p=0.5p=0.5 对称**

pp 和 1−p1−p 容量相同（因为 H2(p)=H2(1−p)H2​(p)=H2​(1−p)）。同理由现象 2。

### 六、计算容量例 3：二元擦除信道（BEC）

擦除信道的容量更直观。

#### 关键观察

接收端要么收到正确比特，要么收到擦除符 ee。当收到 ee 时**完全没信息**；收到正常比特时**完全有信息**（因为 BEC 不会出错，只会丢）。

设输入分布 p(X=0)=qp(X=0)=q，p(X=1)=1−qp(X=1)=1−q。

#### 算 H(X∣Y)H(X∣Y)

- 当 Y≠eY=e（概率 1−ϵ1−ϵ）：完全确定 XX，H(X∣Y=y)=0H(X∣Y=y)=0
- 当 Y=eY=e（概率 ϵϵ）：XX 完全未知，H(X∣Y=e)=H(X)H(X∣Y=e)=H(X)

所以：

H(X∣Y)=(1−ϵ)⋅0+ϵ⋅H(X)=ϵH(X)H(X∣Y)=(1−ϵ)⋅0+ϵ⋅H(X)=ϵH(X)

#### 互信息

I(X;Y)=H(X)−H(X∣Y)=H(X)−ϵH(X)=(1−ϵ)H(X)I(X;Y)=H(X)−H(X∣Y)=H(X)−ϵH(X)=(1−ϵ)H(X)

最大化 H(X)H(X) → 输入均匀 → H(X)=1H(X)=1。

 CBEC=1−ϵ CBEC​=1−ϵ​

#### 直觉的胜利

擦除概率 ϵϵ，容量就损失 ϵϵ。比如丢 30% 的包，剩下 70% 容量。**完全符合直觉**。

但 BSC 不是这样：翻转 30% 的比特，容量只剩 12%（不是 70%）。**为什么？**

因为擦除时**你知道丢了**——可以请求重传或用纠错码精确填补。翻转时**你不知道哪一个错了**——为了纠错要付出更多冗余。"已知的损失"比"未知的损失"温和得多。

### 七、信道容量的几个性质

#### 性质 1：0≤C≤log⁡∣X∣0≤C≤log∣X∣

下界：C=0C=0 当 X,YX,Y 总独立。

上界：C≤min⁡(log⁡∣X∣,log⁡∣Y∣)C≤min(log∣X∣,log∣Y∣)（因为 I(X;Y)≤H(X)≤log⁡∣X∣I(X;Y)≤H(X)≤log∣X∣）。

#### 性质 2：CC 由信道转移概率唯一决定

不依赖于具体使用方式。

#### 性质 3：容量在凸的输入分布集合上取得

I(X;Y)I(X;Y) 作为 p(x)p(x) 的函数是**凹函数**，所以最大值是良定的。优化方法上有标准的 Blahut-Arimoto 算法（迭代算法），不展开。

### 八、几个比较微妙的问题

#### Q1：什么是"一次信道使用"

一个信道符号的传输。BSC 的"一次使用"是发一个比特。AWGN（连续）信道的"一次使用"是发一个实数样本。

实际系统里，"信道使用"对应一个**符号周期**（比如 100 微秒一个 OFDM 符号）。所以"比特/使用"乘以"使用次数/秒"就是"比特/秒"——你 Wi-Fi 路由器上看到的速率。

#### Q2：容量是物理上限还是工程上限

**物理上限**。是数学定理保证的，跟工程实现水平无关。

任何编码方案的传输速率必须 ≤C≤C。这是和热力学第二定律同等地位的硬约束。

#### Q3：能不能"接近"容量

这正是下一节要回答的问题。

**香农第二定理告诉我们**：可以无限接近 CC，但不能超过。具体怎么接近？需要复杂的编码——从 1948 年到 2000 年代发展了 50 多年才在工程上真正接近 CC（Turbo 码、LDPC 码、Polar 码）。

### 九、连续信道：高斯信道（一瞥）

前面讨论的都是离散信道。但你专业里大量信道是**连续**的——电压、电磁波幅度都是实数。

最重要的连续信道模型是**加性高斯白噪声信道**（AWGN）：

Y=X+ZY=X+Z

XX 是发送信号（实数），ZZ 是均值 0、方差 NN 的高斯噪声，独立于 XX。XX 受功率约束 E[X2]≤PE[X2]≤P。

香农推导出（这里只给结论）：

 CAWGN=12log⁡2(1+PN) 比特/使用 CAWGN​=21​log2​(1+NP​) 比特/使用​

考虑带宽 WW（赫兹）后变成更著名的**香农-哈特利公式**：

 C=Wlog⁡2(1+PN0W) 比特/秒 C=Wlog2​(1+N0​WP​) 比特/秒​

这就是 4G、5G、Wi-Fi、卫星通信的理论极限公式。**电子信息工程专业你将来会反复见到它**。

里面的 PN0WN0​WP​ 是**信噪比**（SNR）。SNR 越高、带宽越大，容量越大——非常符合直觉，但定量公式必须靠香农理论。

### 十、本模块小结

**核心定义**：

C=max⁡p(x)I(X;Y)C=p(x)max​I(X;Y)

信道容量 = 在最优输入分布下的最大互信息。**只是信道的属性**。

**三个标准容量**：

|信道|容量|
|---|---|
|无噪二元|C=1C=1|
|BSC（翻转概率 pp）|C=1−H2(p)C=1−H2​(p)|
|BEC（擦除概率 ϵϵ）|C=1−ϵC=1−ϵ|
|AWGN（带宽 WW，SNR）|C=Wlog⁡2(1+SNR)C=Wlog2​(1+SNR)|

**几个直觉**：

- 噪声的随机性才致命，确定性损失可以"算回来"
- 已知错误位置（擦除）比未知错误位置（翻转）好得多
- 容量是物理上限，不能突破，但可以接近

### 接下来

我们已经定义了信道容量 CC。但还有一个关键问题没回答——**CC 是不是真能达到**？

具体地说：

- 在速率 R<CR<C 时，能不能做到几乎无错传输？
- 在速率 R>CR>C 时，会发生什么？
- 怎么做？

这就是**香农第二定理**——**信道编码定理**——信息论中最伟大的结果，也是现代通信工业的理论基石。

它会告诉我们一个让 1948 年的工程师震惊的事实：**只要 R<CR<C，就存在编码方案，让传输错误率任意接近 0**。

这个结论太反直觉了，以至于香农当年也没有给出**怎么构造**这种编码——只证明了**它存在**。从那以后，整整六十年，无数人在寻找"达到香农极限的编码"，直到 2000 年代的 Polar 码才宣告这场长跑的胜利。
