# 11.1 二端口网络

### 一、先聊聊"为什么要有这个概念"

先问你一个问题：

**假设你在设计一个音响系统，信号链是**：

```
  麦克风 → 前级放大 → 均衡器 → 功放 → 喇叭
```

每一级内部都可能有几十上百个元件。**你在分析整个系统时，会去关心"前级放大器"里面每一个晶体管的工作情况吗？**

显然不会。你只关心：

- 给它一个输入电压/电流，输出是多少？
- 它的输入阻抗是多少（决定它怎么"索取"前一级信号）？
- 输出阻抗是多少（决定它怎么"驱动"后一级）？

**换句话说，你把放大器当作一个"黑箱"处理——只关心"端口上的关系"，不关心内部**。

---

**这就是二端口网络的思想**：

> 把一个电路（可以很复杂）抽象成**两对端子**（两个"端口"），只描述端口之间的电压/电流关系。

---

### 二、端口的精确定义

```
   i₁ →                    ← i₂
   ●────────┐     ┌────────●
   │        │     │        │
   + u₁     │  N  │      u₂ +
   -        │     │        -
   ●────────┘     └────────●
   i₁ ←                    → i₂
```

**端口**（port）的严格定义：

> 一对端子，**从一端流入的电流等于从另一端流出**（称为"端口条件"）。

左边两个端子叫**输入端口**（端口 1），右边两个端子叫**输出端口**（端口 2）。

**二端口网络**：有两个端口的线性网络，且**内部不含独立源**。

**四个变量**：u1,i1,u2,i2u1​,i1​,u2​,i2​。其中任选 2 个作**自变量**，另外 2 个作**因变量**——这就是下一节四种参数的由来。
