# 离散数学 第 3 节：真值表、重言式、矛盾式与可满足式

这一节我们要做的事情非常重要：**用一张表，判断任何一个逻辑公式的"性格"**。这是后面所有推理、化简、电路设计的基础工具。

---

### 一、从一个问题开始

我先问你一个问题，你试着凭直觉回答：

> 公式 **P ∨ ¬P** 什么时候是真的？

你只需要考虑两种情况：

- 当 P 为真时，P ∨ ¬P = T ∨ F = **?**
- 当 P 为假时，P ∨ ¬P = F ∨ T = **?**

是不是发现了 —— **无论 P 真假，这个公式永远是真的。**

再看一个：

> **P ∧ ¬P** 什么时候是真的？

- P 为真：T ∧ F = **F**
- P 为假：F ∧ T = **F**

**永远是假的。**

你看，逻辑公式其实可以分成三种"性格"：

- 有些公式**永远真**（像 P ∨ ¬P）
- 有些公式**永远假**（像 P ∧ ¬P）
- 有些公式**有时真有时假**（像 P → Q）

这三种性格，就是我们这节要学的**三个核心概念**。但在定义它们之前，我们要先掌握一个工具 —— **真值表**。

---

### 二、真值表 (Truth Table)

#### 2.1 基本思想

真值表的想法简单得有点不讲道理：

> **一个公式里有 n 个原子命题，那就把所有 2ⁿ 种真假组合都列出来，挨个算公式的值。**

这和电子信息里的"穷举所有输入组合测试电路"完全是一回事。

#### 2.2 为什么是 2ⁿ？

先问你一个小问题：

> 如果公式里有 3 个原子命题 P、Q、R，一共有多少种真假组合？

想一下再看答案。

---

每个命题独立有 2 种取值 (T/F)，3 个命题就是 2 × 2 × 2 = **8 种**。一般地，n 个命题有 **2ⁿ** 种组合。

这也是为什么 n 大的时候真值表会爆炸 —— 10 个命题就有 1024 行了。不过做习题时 n 通常不超过 3 或 4，手算完全可行。

#### 2.3 怎么列真值表？一个标准流程

我们以公式 **(P → Q) ∧ ¬Q** 为例，一步一步列出来。

**第一步：确定列数**

列真值表时，除了原子命题列，还要**列出所有中间子公式**，最后算主公式。这么做是为了**不出错**，而不是为了好看。

对于 (P → Q) ∧ ¬Q，我需要的列是：

|P|Q|P → Q|¬Q|(P → Q) ∧ ¬Q|
|---|---|---|---|---|

**第二步：填原子命题的所有组合**

有个小技巧让你不漏不重：**左边一列填 TTFF，右边一列填 TFTF**（像二进制计数）。

|P|Q|
|---|---|
|T|T|
|T|F|
|F|T|
|F|F|

如果有 3 个原子命题 P、Q、R，就是 P 填 TTTTFFFF，Q 填 TTFFTTFF，R 填 TFTFTFTF。**永远这个顺序，养成习惯。**

**第三步：逐列计算**

回忆一下上一节的 → 真值表：**只有 T → F 是假，其他都是真**。

|P|Q|P → Q|
|---|---|---|
|T|T|T|
|T|F|**F**|
|F|T|T|
|F|F|T|

然后 ¬Q 就是把 Q 列反过来：

|P|Q|P → Q|¬Q|
|---|---|---|---|
|T|T|T|F|
|T|F|F|T|
|F|T|T|F|
|F|F|T|T|

最后算 (P → Q) ∧ ¬Q，就是把"P→Q"这一列和"¬Q"这一列做 ∧ 运算（两个都 T 才 T）：

|P|Q|P → Q|¬Q|(P → Q) ∧ ¬Q|
|---|---|---|---|---|
|T|T|T|F|F|
|T|F|F|T|F|
|F|T|T|F|F|
|F|F|T|T|**T**|

看最后一列 —— 这个公式**只在 P=F, Q=F 时为真**，其他情况都为假。

#### 🤔 轮到你了

请你**自己动手**列出下面这个公式的真值表：

(P∨Q)→¬P(P \lor Q) \to \neg P(P∨Q)→¬P

按照我上面的格式，至少要列出 P、Q、P∨Q、¬P、最终公式 这 5 列。做完在心里对一下答案（或者写下来），然后继续往下看。

---

**参考答案**：

|P|Q|P ∨ Q|¬P|(P ∨ Q) → ¬P|
|---|---|---|---|---|
|T|T|T|F|F|
|T|F|T|F|F|
|F|T|T|T|T|
|F|F|F|T|T|

重点检查：第 1、2 行 P=T，所以 ¬P=F。P∨Q 都是 T，T→F = **F**。对上了吗？

### 三、三种"性格"：重言式、矛盾式、可满足式

现在我们可以正式给三个名字了。

#### 3.1 重言式 (Tautology)

> **定义**：一个命题公式，**在任何真值指派下都为真**，称为重言式，也叫**永真式**。

通俗说：真值表的最后一列**全是 T**。

**符号**：有时写作 ⊨ A，意思是"A 是重言式"。

**经典例子**：

**例 1**：P ∨ ¬P（排中律）

|P|¬P|P ∨ ¬P|
|---|---|---|
|T|F|T|
|F|T|T|

**例 2**：(P ∧ Q) → P

先问你：这个公式在说什么？用自然语言讲就是——"**如果 P 和 Q 都成立，那么 P 成立**"。听起来是不是废话一样正确？让我们验证：

|P|Q|P ∧ Q|(P ∧ Q) → P|
|---|---|---|---|
|T|T|T|T|
|T|F|F|T|
|F|T|F|T|
|F|F|F|T|

全是 T，确实是重言式。注意第 3、4 行：P∧Q 为假，F→任何 都是 T（还记得蕴含的"空虚为真"吗？）。

**例 3**（稍微复杂一点）：((P → Q) ∧ P) → Q（**假言推理**，后面讲推理规则时会用到）

|P|Q|P → Q|(P → Q) ∧ P|((P → Q) ∧ P) → Q|
|---|---|---|---|---|
|T|T|T|T|T|
|T|F|F|F|T|
|F|T|T|F|T|
|F|F|T|F|T|

也是永真。这个公式表达的是："**如果 P 能推出 Q，并且 P 成立，那么 Q 成立**" —— 这正是数学证明里最基础的推理方式。

#### 3.2 矛盾式 (Contradiction)

> **定义**：一个命题公式，**在任何真值指派下都为假**，称为矛盾式，也叫**永假式**。

真值表最后一列**全是 F**。

**经典例子**：

**例 1**：P ∧ ¬P（矛盾律）

|P|¬P|P ∧ ¬P|
|---|---|---|
|T|F|F|
|F|T|F|

**例 2**：(P → Q) ∧ P ∧ ¬Q

这个式子说："P 推出 Q，P 成立，但 Q 不成立"。直觉上这不可能，对吗？我们验证：

|P|Q|P → Q|¬Q|(P → Q) ∧ P ∧ ¬Q|
|---|---|---|---|---|
|T|T|T|F|F|
|T|F|F|T|F|
|F|T|T|F|F|
|F|F|T|T|F|

全是 F。这个公式其实就是**反证法的逻辑依据**：如果你能从"P 推出 Q"和"P 真"推出"Q 假"，那就自相矛盾，说明假设错了。

#### 3.3 可满足式 (Satisfiable)

> **定义**：一个命题公式，**至少存在一种真值指派使它为真**，称为可满足式。

真值表最后一列**至少有一个 T** 就行。

**注意关系**：

- 重言式一定是可满足式（全 T 当然包含至少一个 T）。
- 矛盾式一定**不是**可满足式。
- 不是重言式也不是矛盾式的公式，叫**偶然式 (contingency)**，它必然是可满足式。

可以画个图来理清关系：

```
所有命题公式
├── 重言式 (全真)     ← 是可满足式
├── 偶然式 (有真有假) ← 是可满足式  
└── 矛盾式 (全假)     ← 不是可满足式
```

#### 🤔 检验理解

你来判断下面三个公式分别是哪种"性格"（先不要列真值表，凭直觉猜一下，再列表验证）：

1. (P → Q) ↔ (¬Q → ¬P)
2. P ∧ (¬P ∨ Q)
3. (P ∨ Q) ∧ (¬P ∧ ¬Q)

---

我们一起验证第 1 个，剩下两个你自己做：

**公式 1**：(P → Q) ↔ (¬Q → ¬P)

|P|Q|P → Q|¬P|¬Q|¬Q → ¬P|(P→Q) ↔ (¬Q→¬P)|
|---|---|---|---|---|---|---|
|T|T|T|F|F|T|T|
|T|F|F|F|T|F|T|
|F|T|T|T|F|T|T|
|F|F|T|T|T|T|T|

**全 T，是重言式。**

这个公式特别重要，它说的是 **"P → Q" 等价于 "¬Q → ¬P"**。后者叫做前者的**逆否命题 (contrapositive)**。这是数学证明里的一个黄金工具 —— 有时候正着证难，反着证（证逆否）就简单了。

例子：要证明"**如果 n² 是偶数，那么 n 是偶数**"。

- 正着很难：从 n² 是偶数反推 n 的性质。
- 证逆否"**如果 n 是奇数，那么 n² 是奇数**"就简单了：n = 2k+1，n² = 4k² + 4k + 1 = 2(2k²+2k) + 1，是奇数。

### 四、可满足性问题：一个工程师必须知道的背景

既然提到了可满足式，顺便给你讲一件事，这对你（电子信息工程）未来很重要：

> **SAT 问题 (Satisfiability Problem)**：给定一个命题公式，判断它是否可满足。

听起来很简单对吧？但这是**计算机科学中最著名的 NP 完全问题** —— 也就是说，当公式里的命题数 n 很大时，目前**找不到多项式时间的算法**能普遍解决它。当 n = 100，真值表就有 2¹⁰⁰ ≈ 10³⁰ 行，再快的计算机也算不完。

这个看似抽象的问题，实际上是：

- **芯片验证**：一个数字电路设计是否满足规范？ → 转化为 SAT 问题
- **自动定理证明**：AI 能不能自动找证明？ → 核心是 SAT
- **密码学**：很多密码方案的安全性就建立在 SAT 的困难性上

所以你现在学的这张真值表，在工业界对应的是 **SAT solver**，Intel、Synopsys 这些公司的 EDA 软件里天天在跑。

### 五、一些常见的重言式（值得记住）

这些在后面化简和证明时会反复用到，**现在不用背，有个印象即可**，用多了自然记住：

|名称|公式|
|---|---|
|排中律|P ∨ ¬P|
|矛盾律（的否定）|¬(P ∧ ¬P)|
|同一律|P → P|
|双重否定|P ↔ ¬¬P|
|逆否等价|(P → Q) ↔ (¬Q → ¬P)|
|假言推理|((P → Q) ∧ P) → Q|
|拒取式|((P → Q) ∧ ¬Q) → ¬P|
|假言三段论|((P → Q) ∧ (Q → R)) → (P → R)|

**假言三段论**是不是很眼熟？"如果 P 推出 Q，Q 又推出 R，那 P 推出 R"—— 整个数学证明的链条就是靠它串起来的。

### 练习 1：判断公式类型

#### (1) (P → Q) ∨ (Q → P)

这个**反直觉**的点在于："P 推 Q" 和 "Q 推 P" 至少有一个成立 —— 听起来好像不一定？我们列真值表看看：

|P|Q|P → Q|Q → P|(P → Q) ∨ (Q → P)|
|---|---|---|---|---|
|T|T|T|T|T|
|T|F|F|T|T|
|F|T|T|F|T|
|F|F|T|T|T|

**全 T，重言式。**

**为什么反直觉？** 因为我们常默认 P 和 Q 有某种"因果关系"。但在形式逻辑里，只要 P 为假，"P → Q" 自动为真；只要 Q 为真，"Q → P" 自动为真。所以**至少一个成立**是必然的。这也是一个警告：**形式逻辑的蕴含 ≠ 日常语言的"因果"**。

#### (2) (P ∧ Q) ∧ ¬(P ∨ Q)

|P|Q|P ∧ Q|P ∨ Q|¬(P ∨ Q)|(P ∧ Q) ∧ ¬(P ∨ Q)|
|---|---|---|---|---|---|
|T|T|T|T|F|F|
|T|F|F|T|F|F|
|F|T|F|T|F|F|
|F|F|F|F|T|F|

**全 F，矛盾式。**

直觉解释：公式说"P 和 Q 都真，同时 P 和 Q 都不真"，显然自相矛盾。

#### (3) (P → Q) → (P ∧ Q)

|P|Q|P → Q|P ∧ Q|(P → Q) → (P ∧ Q)|
|---|---|---|---|---|
|T|T|T|T|T|
|T|F|F|F|T|
|F|T|T|F|**F**|
|F|F|T|F|**F**|

**有 T 有 F，可满足但非重言。**

关键行是第 3 行：P=F, Q=T 时，P→Q 为真（前件假，空虚为真），但 P∧Q 为假，所以 T→F = F。

#### (4) ((P → Q) ∧ (Q → R)) → (P → R) 假言三段论

这题有 3 个命题，8 行：

|P|Q|R|P→Q|Q→R|(P→Q)∧(Q→R)|P→R|主公式|
|---|---|---|---|---|---|---|---|
|T|T|T|T|T|T|T|T|
|T|T|F|T|F|F|F|T|
|T|F|T|F|T|F|T|T|
|T|F|F|F|T|F|F|T|
|F|T|T|T|T|T|T|T|
|F|T|F|T|F|F|T|T|
|F|F|T|T|T|T|T|T|
|F|F|F|T|T|T|T|T|

**全 T，重言式。** 这就是**假言三段论**，数学证明链的理论基础。

### 练习 2：A = (P ∨ ¬Q) → R

#### (1) 真值表

|P|Q|R|¬Q|P ∨ ¬Q|(P ∨ ¬Q) → R|
|---|---|---|---|---|---|
|T|T|T|F|T|T|
|T|T|F|F|T|**F**|
|T|F|T|T|T|T|
|T|F|F|T|T|**F**|
|F|T|T|F|F|T|
|F|T|F|F|F|T|
|F|F|T|T|T|T|
|F|F|F|T|T|**F**|

#### (2) 可满足吗？

**是可满足式**。比如 P=T, Q=T, R=T（第 1 行）就让它为真。

#### (3) 重言式吗？

**不是**。第 2、4、8 行为 F，存在反例。

### 练习 3：电子电路

#### (1) 逻辑表达式

"A 为真" 且 "B 和 C 不同时为真"：

F=A∧¬(B∧C)F = A \land \neg(B \land C)F=A∧¬(B∧C)

#### (2) 真值表

|A|B|C|B ∧ C|¬(B ∧ C)|F = A ∧ ¬(B ∧ C)|
|---|---|---|---|---|---|
|T|T|T|T|F|F|
|T|T|F|F|T|**T**|
|T|F|T|F|T|**T**|
|T|F|F|F|T|**T**|
|F|T|T|T|F|F|
|F|T|F|F|T|F|
|F|F|T|F|T|F|
|F|F|F|F|T|F|

#### (3) 类型

**可满足式**（有 T 有 F，偶然式）。

💡 **工程小知识**：A ∧ ¬(B ∧ C) 在数字电路里就是 **AND 门 + NAND 门**。这个电路模式经常出现在"使能信号 + 互斥保护"的场景，比如防止两个功率管同时导通。

### 练习 4：思考题

> "如果 1+1=3，那么我就是总统" —— 按逻辑这句话是真的，但这荒谬吗？

**我的回答思路**：

这个"荒谬感"其实源于**混淆了两种"真"**：

1. **形式逻辑的真**：只关心真值表。前件假时蕴含整体为真，这是**定义**，为的是保证逻辑系统的一致性和简洁性。
2. **日常语言的"合理"**：通常暗含"前件和后件有意义的因果关系"。

数学家选择"前件假则整体真"的定义，是因为这样做：

- 让很多重要命题（如"对任意 x>0 有 x²>0"）能被统一表达 —— 当 x≤0 时我们希望蕴含"自动成立"。
- 保证了像 (P → Q) ↔ (¬Q → ¬P) 这样的等价关系成立。

所以那句话**在形式逻辑里是真的，但在日常交流中是无意义的**，因为它的前件永远假，没有传递任何信息。**逻辑的"真"是一种技术性的真，不等于"有意义"**。
