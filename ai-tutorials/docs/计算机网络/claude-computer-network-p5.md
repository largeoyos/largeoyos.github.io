# 第四章 网络层

网络层是整个计算机网络的**核心**,也是考研和期末考的**重中之重**。子网划分几乎 100% 会考,IP 分组转发、路由算法也是大题常客。这章内容多,我分几节讲透。

---

### 第一节 网络层概述

#### 1.1 网络层的功能(**必背**)

1. **异构网络互连**:把不同类型的网络连起来(通过路由器)
2. **路由与转发**
    - **路由(Routing)**:选择路径的过程(路由协议,慢)
    - **转发(Forwarding)**:根据路由表把分组送到下一跳(快)
3. **拥塞控制**:防止网络过载

📌 **路由 vs 转发**(易错辨析):

- 路由是**全局**的,动态构建路由表
- 转发是**局部**的,每个路由器根据自己的路由表做一次决策

#### 1.2 两种服务方式

||虚电路服务|数据报服务|
|---|---|---|
|思想|先建立连接再发送|无连接,每个分组独立路由|
|目的地址|仅建立阶段需要|**每个分组都需要完整地址**|
|分组顺序|**按序到达**|可能失序|
|可靠性|网络保证|端系统保证|
|代表|X.25、ATM|**IP(互联网用的就是这个)**|

📌 **一句话结论**:**互联网是数据报网络**,IP 协议提供**无连接、不可靠、尽力而为**的服务。

### 第二节 IPv4(重点中的重点)

#### 2.1 IPv4 地址结构

**32 位**,采用**点分十进制**表示,如 `192.168.1.1`。

232≈43 亿,已经不够用,所以推 IPv62^{32} \approx 43 \text{ 亿},\text{已经不够用,所以推 IPv6}232≈43 亿,已经不够用,所以推 IPv6

#### 2.2 分类 IP 地址(**历史但必考**)

IP 地址 = **网络号 + 主机号**

|类|首位|网络号位数|主机号位数|范围|用途|
|---|---|---|---|---|---|
|**A**|0|8|24|1.x.x.x ~ 126.x.x.x|大型网络|
|**B**|10|16|16|128.x.x.x ~ 191.x.x.x|中型网络|
|**C**|110|24|8|192.x.x.x ~ 223.x.x.x|小型网络|
|**D**|1110|—|—|224.x.x.x ~ 239.x.x.x|**多播**|
|**E**|11110|—|—|240.x.x.x ~ 255.x.x.x|保留|

📌 **快速判断技巧**:看第一个字节的十进制数。

- 1~126 → A 类
- 128~191 → B 类
- 192~223 → C 类

#### 2.3 特殊 IP 地址(**必背**)

|主机号|网络号|作为源地址|作为目的地址|含义|
|---|---|---|---|---|
|全 0|全 0|✓|✗|**本网络本主机**(启动时用)|
|全 0|特定值|✗|✗|**网络地址**(表示某个网络)|
|全 1|全 1|✗|✓|**受限广播**(仅本网络)|
|全 1|特定值|✗|✓|**直接广播**(发给某网络所有主机)|
|127|任意(非全 0 全 1)|✓|✓|**环回地址**(测试本机)|

📌 **最常考**:

- `127.0.0.1`:本机环回
- `255.255.255.255`:受限广播
- `0.0.0.0`:本机

#### 2.4 私有 IP 地址(**必背**)

以下地址**不在公网路由**,供内部使用:

|类|范围|
|---|---|
|A|`10.0.0.0 ~ 10.255.255.255`|
|B|`172.16.0.0 ~ 172.31.255.255`|
|C|`192.168.0.0 ~ 192.168.255.255`|

你家里 WiFi 分配的 `192.168.x.x` 就是私有地址,通过 **NAT** 转换后才能上公网。

### 第三节 子网划分(**必考计算题**)

#### 3.1 为什么要子网划分

分类 IP 地址浪费严重:一个 B 类网络有 216=655342^{16} = 65534 216=65534 个主机号,实际一个公司用不了这么多。

**解决**:借**主机号的高若干位作为子网号**,把一个大网分成多个小网。

#### 3.2 子网掩码(**核心概念**)

- **32 位**,由连续的 1(网络位+子网位)和连续的 0(主机位)组成
- **作用**:将 IP 地址**按位与**子网掩码,得到**网络地址**

📌 **默认子网掩码**:

- A 类:`255.0.0.0` = `/8`
- B 类:`255.255.0.0` = `/16`
- C 类:`255.255.255.0` = `/24`

#### 3.3 子网划分计算(**必会**)

**例 1**:将 C 类网络 `192.168.1.0/24` 划分为 4 个子网。

**分析**:

- 需要 log⁡24=2\log_2 4 = 2 log2​4=2 位作子网号
- 借 2 位后,新掩码 = 24 + 2 = **/26** = `255.255.255.192`
- 每个子网主机号 = 32 - 26 = 6 位,可分 26−2=622^6 - 2 = 62 26−2=62 台主机

**四个子网**:

|子网号|子网地址|地址范围|广播地址|
|---|---|---|---|
|00|192.168.1.0/26|.1 ~ .62|.63|
|01|192.168.1.64/26|.65 ~ .126|.127|
|10|192.168.1.128/26|.129 ~ .190|.191|
|11|192.168.1.192/26|.193 ~ .254|.255|

📌 **两个必扣**:

- **网络地址**(主机号全 0)**不能分配给主机**
- **广播地址**(主机号全 1)**不能分配给主机**
- 所以每个子网**可用主机数 = 2n−22^n - 2 2n−2**(nn n 为主机号位数)

#### 3.4 子网划分例题套路(**必须熟练**)

**题目**:给你 IP `172.16.20.200/20`,求网络地址、广播地址、可用地址范围、主机数。

**解法(通用 4 步)**:

**第 1 步:写出掩码** /20 = `11111111.11111111.11110000.00000000` = `255.255.240.0`

**第 2 步:IP 与 掩码 按位与 → 网络地址**

IP: `172.16.00010100.11001000` 掩码:`255.255.11110000.00000000` 与: `172.16.00010000.00000000` = **`172.16.16.0`**

**第 3 步:网络地址 + 主机号全 1 → 广播地址** 主机号 12 位全 1: `172.16.00011111.11111111` = **`172.16.31.255`**

**第 4 步:可用范围** **`172.16.16.1 ~ 172.16.31.254`**主机数 = 212−2=40942^{12} - 2 = 4094 212−2=4094

📌 **技巧**:记住"**块大小**"法——第三字节每 16 为一块(因为借了 4 位,256/24=16256/2^4 = 16 256/24=16),所以 20 落在 `[16, 32)` 块内,网络地址第三字节 = 16,广播地址第三字节 = 31。

#### 3.5 子网划分考题常见陷阱

1. **最后一个子网地址的主机号可不可以全 1?**
    - 旧规定:**不能用全 0 和全 1 的子网号**
    - 新规定(CIDR):**都可以用**
    - 考试看题目是否明确说明,**408 默认允许**
2. **给定主机数,求最少借几位?**
    - 例:需要 30 台主机 → 主机位至少 5 位(25−2=302^5-2=30 25−2=30)
    - 给的是 C 类,则子网位 = 8-5 = 3 位,可划分 23=82^3 = 8 23=8 个子网

### 第四节 CIDR(无分类域间路由选择,**必考**)

#### 4.1 CIDR 核心思想

**消除 A/B/C 类划分,用"斜线记法"**:`a.b.c.d/n`,n 为网络前缀长度。

例:`128.14.32.0/20` 表示前 20 位是网络前缀。

#### 4.2 CIDR 地址块

一个 CIDR 地址块 = **连续的一组 IP 地址**,共 232−n2^{32-n} 232−n 个。

#### 4.3 路由聚合(构造超网)(**必考**)

把多个**小网络**合并为一个**大网络**的路由表项,减小路由表规模。

**例题**(经典):将以下 4 个地址块聚合:

- `206.1.0.0/17`
- `206.1.128.0/17`
- `206.2.0.0/17`
- `206.2.128.0/17`

**解法**:写成二进制看公共前缀长度。

前两个 → `206.1.0.0/16` 后两个 → `206.2.0.0/16`

再聚合: `206.1.0.0/16` = `11001110.00000001.xxx` `206.2.0.0/16` = `11001110.00000010.xxx`

前 15 位相同 → 聚合为 **`206.0.0.0/15`**

📌 **口诀**:找最长公共前缀。

#### 4.4 最长前缀匹配(**转发核心原则**)

路由器查表时,如果有多条路由匹配同一目的地址,**选择前缀最长的那一条**(最具体的路由)。

#### 5.1 重点字段解析

|字段|长度|作用|
|---|---|---|
|**版本**|4 位|IPv4 = 4,IPv6 = 6|
|**首部长度**|4 位|以 **4 字节** 为单位,最小 5(20 字节),最大 15(60 字节)|
|**总长度**|16 位|整个数据报长度,最大 216−1=655352^{16}-1 = 65535 216−1=65535 字节|
|**标识**|16 位|分片时同一数据报的所有片用相同标识|
|**标志**|3 位|MF(更多分片)、DF(不分片)|
|**片偏移**|13 位|分片在原报文中的位置,以 **8 字节** 为单位|
|**TTL**|8 位|跳数,每经过一个路由器 **-1**,为 0 则丢弃|
|**协议**|8 位|上层协议(TCP=6,UDP=17,ICMP=1)|
|**首部检验和**|16 位|只校验首部,**不校验数据**(每经一个路由器要**重新计算**)|

📌 **高频考点**:

- **TTL** 作用:防止分组在网络中无限循环
- **协议字段**数字:**ICMP=1,TCP=6,UDP=17**(背!)
- 首部检验和为什么只校验首部?——**数据校验交给传输层做,避免重复**

#### 5.2 IP 分片(**必考计算**)

当 IP 数据报**大于链路 MTU** 时需要分片。

**分片规则**:

- 每片的**数据部分长度**必须是 **8 的倍数**(片偏移单位是 8 字节)
- 除最后一片外,**MF = 1**(还有更多片),最后一片 **MF = 0**
- 所有分片的 **标识相同**
- **片偏移 = 该片数据第一个字节在原报文数据中的位置 / 8**

**经典例题**: 原 IP 数据报总长 4000 字节(首部 20 + 数据 3980),MTU = 1500,求分片结果。

**解**:每片数据部分 ≤ 1500 - 20 = 1480,且必须是 8 的倍数。 1480 恰好是 8 的倍数 ✓

|片号|数据长度|总长度|MF|片偏移|
|---|---|---|---|---|
|1|1480|1500|1|0|
|2|1480|1500|1|185(=1480/8)|
|3|1020|1040|0|370(=2960/8)|

验证:1480+1480+1020=39801480+1480+1020=3980 1480+1480+1020=3980 ✓

📌 **易错点**:

- **总长度** = 首部 + 数据,**不是单独的数据长度**
- **片偏移 = 数据字节位置 / 8**,注意是**数据部分**的位置

### 第六节 ARP 协议(**必考**)

#### 6.1 为什么需要 ARP?

同一局域网内,发送数据**最终需要目的 MAC 地址**(数据链路层才能成帧)。ARP 就是**根据 IP 地址查询 MAC 地址**。

#### 6.2 ARP 工作过程

1. 主机 A 想发数据给主机 B,只知 B 的 IP
2. A 查自己的 **ARP 缓存**,找 B 的 MAC
3. 没找到 → A **广播 ARP 请求**:"IP 是 X 的主机,告诉我你的 MAC!"
4. 局域网内所有主机收到,只有 B 回应 **ARP 应答**(单播):"我的 MAC 是 Y"
5. A 把 (IP, MAC) 存入 **ARP 缓存**(有时效性,通常 10-20 分钟)

📌 **考点归纳**:

- ARP **请求是广播,应答是单播**
- ARP 工作在 **网络层**(也有说法认为在链路层,**408 认定为网络层**)
- **ARP 只在同一局域网内有效**,跨网段需要通过默认网关

#### 6.3 跨网段通信中的 ARP(**极重要**)

A(192.168.1.2)要发数据给 B(10.0.0.2),它们不在同一网段。

流程:

1. A 比较自己和 B 的网络号,**发现不在同一网段**
2. A 把数据的**目的 IP = B 的 IP**,但**目的 MAC = 默认网关的 MAC**(通过 ARP 查网关的 MAC)
3. 数据到路由器,路由器查路由表,**修改 MAC 头**(源 MAC = 路由器出接口 MAC,目的 MAC = 下一跳或 B 的 MAC),**IP 头不变**
4. 一路转发到 B

📌 **核心规律**:

- **IP 地址在传输过程中不变**(源 IP、目的 IP 保持不变)
- **MAC 地址每经过一个路由器就会改变**(每一跳都是局域网内通信)

### 第七节 DHCP、ICMP、NAT

#### 7.1 DHCP(动态主机配置协议)

**功能**:自动给主机分配 IP 地址等配置信息。

**工作过程(DORA,四步)**:

1. **Discover**:客户端广播 "我要 IP"
2. **Offer**:DHCP 服务器广播 "给你这个 IP"
3. **Request**:客户端广播 "我要接受这个 IP"
4. **Ack**:服务器广播 "确认"

📌 全程使用**广播**,因为客户端**还没有 IP**。使用 **UDP**,服务器端口 67,客户端端口 68。

#### 7.2 ICMP(网际控制报文协议)

**功能**:在 IP 网络中传递**差错和控制信息**。

**ICMP 报文类型**:

- **差错报告**:终点不可达、超时、参数问题、改变路由、源点抑制
- **询问**:**回送请求/应答(ping 用)**、时间戳请求/应答

**两大著名应用**:

- **ping**:用 ICMP 回送请求/应答测试连通性
- **traceroute(tracert)**:用**TTL 递增**的 IP 数据报,每一跳返回 ICMP 超时报文,从而发现路径

📌 **考点**:

- ping = ICMP(不是 TCP 也不是 UDP)
- ICMP 封装在 **IP 数据报的数据部分**,**协议号 = 1**

#### 7.3 NAT(网络地址转换)

**功能**:私有 IP ↔ 公网 IP 转换,缓解 IPv4 地址枯竭。

**三种类型**:

- **静态 NAT**:一对一固定映射
- **动态 NAT**:多对多动态映射
- **NAPT(最常用)**:**多对一 + 端口号**,一个公网 IP 通过不同端口号映射多个内部主机(你家路由器用的就是这个)

### 第八节 路由算法与路由协议

#### 8.1 路由算法分类

- **静态路由**:人工配置,适合小网络
- **动态路由**:路由器之间交换信息,自动更新

#### 8.2 自治系统 AS

**自治系统 AS**:一个在统一管理下的路由器集合。互联网由众多 AS 组成。

路由协议分两大类:

- **内部网关协议 IGP**:AS 内部用(**RIP、OSPF**)
- **外部网关协议 EGP**:AS 之间用(**BGP**)

#### 8.3 RIP(距离向量协议,**必考**)

- 算法:**Bellman-Ford**(距离向量)
- 度量:**跳数(hop count)**,最大 15,**16 表示不可达**
- 更新:**每 30 秒**向邻居广播自己的**整张路由表**
- **只与相邻路由器交换信息**

**距离向量算法核心**:

Dx(y)=min⁡v{c(x,v)+Dv(y)}D_x(y) = \min_v \{c(x,v) + D_v(y)\}Dx​(y)=vmin​{c(x,v)+Dv​(y)}

意思:x 到 y 的最短距离 = 相邻节点 v 到 y 的距离 + x 到 v 的代价,取最小。

**RIP 更新规则**:

- 收到邻居 X 的路由表,所有表项跳数 +1
- 若表中有新目的地 → 加入
- 若表中已有目的地但来自 X → **无条件更新**(信任老朋友)
- 若表中已有目的地来自其他 → **比较跳数**,小则更新,大则不变

📌 **缺点**:**收敛慢,易产生路由环路**(好消息传得快,坏消息传得慢)

#### 8.4 OSPF(链路状态协议,**必考**)

- 算法:**Dijkstra**(链路状态)
- 度量:**链路代价**(可基于带宽、延迟)
- 更新:**洪泛法**向**所有**路由器发送**链路状态**(不是整张路由表)
- **只在链路状态变化时发送**
- 基于 IP,**协议号 89**

**OSPF 与 RIP 对比**(**必考辨析**):

||RIP|OSPF|
|---|---|---|
|算法|距离向量|链路状态|
|发送内容|**整张路由表**|**链路状态**|
|发送对象|**相邻路由器**|**AS 内所有路由器**|
|发送时机|周期性(30s)|状态变化时|
|度量|跳数|链路代价|
|适用规模|小型|大型|
|封装|UDP|直接在 IP 上(协议号89)|

#### 8.5 BGP(外部网关协议)

- **路径向量协议**
- AS 之间使用
- **基于 TCP**(端口 179)
- 目标是**找到一条可达路径**,不一定最优(涉及策略、商业利益)

### 第九节 IPv6(了解)

#### 9.1 为什么需要 IPv6

- IPv4 地址只有 32 位,已枯竭
- IPv6 使用 **128 位**,地址数 21282^{128} 2128

#### 9.2 IPv6 地址表示

- **冒分十六进制**:`2001:0DB8:0000:0000:0000:0000:1428:57AB`
- **零压缩**:连续的 0 可用 `::` 表示(**只能用一次**)
- 例:上面地址 → `2001:DB8::1428:57AB`

#### 9.3 IPv6 改进(必背)

- 地址空间:32 → **128 位**
- 首部:**定长 40 字节**(IPv4 变长)
- **取消首部检验和**(交给上层)
- **取消分片字段**(由源主机完成路径 MTU 发现)
- **支持即插即用**(自动配置)
- **内置 IPsec**(安全性)

### 第十节 路由器

#### 10.1 路由器工作原理

1. 收到 IP 数据报,拆到网络层
2. 查**路由表**,决定下一跳
3. 重新封装链路层帧(**源 MAC 改为出接口 MAC,目的 MAC 改为下一跳 MAC**)
4. **TTL -1,重新计算首部检验和**
5. 发往下一跳

📌 **路由器不修改**:源 IP、目的 IP(NAT 除外)

📌 **路由器修改**:MAC 地址、TTL、首部检验和

#### 10.2 路由表基本结构

|目的网络|子网掩码|下一跳|接口|
|---|---|---|---|
|192.168.1.0|255.255.255.0|直连|eth0|
|10.0.0.0|255.0.0.0|172.16.1.1|eth1|
|0.0.0.0|0.0.0.0|172.16.1.1|eth1|

- `0.0.0.0/0`:**默认路由**(其他都不匹配时走这个)

#### 10.3 分组转发算法

```
1. 提取目的 IP 地址 D
2. 对路由表中每一条记录,把 D 与掩码做 AND,若结果 = 该记录的目的网络,则匹配
3. 若有多条匹配,选择最长前缀匹配
4. 若未找到,使用默认路由
5. 若无默认路由,丢弃并发送 ICMP 不可达
```
