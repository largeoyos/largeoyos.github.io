# 第六章 应用层

最后一章了,内容相对轻松但考点不少。在正式开讲之前,我想先用一个**小测试**了解你当前的基础——这能让我把讲解的重点放在你真正需要的地方,而不是泛泛地铺开。

---

### 📋 学前自测(5 个小问题,凭直觉答)

**问题 1**:你在浏览器地址栏输入 `www.baidu.com` 并回车,大致发生了什么?能说出几个关键步骤就行,不需要精确。

**问题 2**:HTTP 是"无状态"的,你能猜到这句话是什么意思吗?那为什么网站还能"记住"你已登录?

**问题 3**:你知道 HTTP 和 HTTPS 的区别吗?多一个 S 多了什么?

**问题 4**:发邮件和收邮件,用的是同一个协议吗?你用过什么邮件客户端(QQ 邮箱 / Outlook / 网易)?

**问题 5**:"DNS 劫持""CDN 加速""Cookie""缓存"这些词,哪些你听过但不太清楚?

---

### 🤔 或者,你也可以直接选一个切入方式

如果你觉得逐个回答麻烦,也可以直接告诉我想从哪个角度切入:

**A. 按教材顺序讲**——DNS → FTP → 邮件 → WWW/HTTP → P2P,系统全面,适合应试

**B. 跟着"访问一个网页"的流程讲**——从你敲下 URL 到页面显示,所有协议串起来讲,适合理解

**C. 按考点密度讲**——先讲 HTTP、DNS 这两个大头,其他快速过,适合考前突击

---

你是大一学生,我更建议你选 **B**——因为你已经学了前面五章,这里正好是一次**总复习式的串讲**,把 HTTP、DNS、TCP、IP、ARP、以太网全部串起来,一口气理解"互联网是怎么工作的"。这种整体感,比背一堆零散的协议重要得多。

不过最终你来选。**你想从哪里开始?** 或者先回答上面几个自测题,我根据你的答案调整讲法。
