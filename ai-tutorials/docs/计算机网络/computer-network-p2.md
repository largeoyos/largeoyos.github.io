# 第一章:概述与体系结构

---

### 第一节 计算机网络的定义与组成

#### 1.1 定义(考点:名词解释)

**计算机网络**是指将地理位置不同的、具有独立功能的多台计算机及其外部设备,通过通信线路连接起来,在网络操作系统、网络管理软件及网络通信协议的管理和协调下,实现资源共享和信息传递的计算机系统。

这个定义里有四个关键词,考试常考:

1. **地理位置不同**——强调分布性
2. **独立功能**——每台计算机自身是完整的(区别于"多终端系统")
3. **通信线路连接**——有物理连接
4. **协议**——按规则通信(这是网络和"一堆电脑堆在一起"的根本区别)

#### 1.2 组成(三种划分方式,都可能考)

**按工作方式分**:

- **边缘部分**:所有连接在互联网上的主机,用户直接使用(端系统)
- **核心部分**:由大量网络和连接这些网络的路由器组成,为边缘部分提供服务

**按功能组成分**:

- **通信子网**:物理层、数据链路层、网络层设备(实现数据传输)
- **资源子网**:主机、终端、软件、数据(实现资源共享)

**按组成部分分**:硬件、软件、协议

📌 **易错点**:路由器属于通信子网/核心部分,主机属于资源子网/边缘部分。

### 第二节 计算机网络的功能

记住五个,按重要性排序:

1. **数据通信**(最基本)
2. **资源共享**(硬件、软件、数据)
3. **分布式处理**
4. **提高可靠性**(一台挂了还有其他的)
5. **负载均衡**

---

### 第三节 计算机网络的分类(考点密集)

#### 3.1 按分布范围分(最常考)

|类型|全称|范围|举例|
|---|---|---|---|
|PAN|个人区域网|1~10m|蓝牙|
|LAN|局域网|10m~1km|校园网、办公室|
|MAN|城域网|5~50km|城市宽带|
|WAN|广域网|>50km|互联网骨干|

📌 **易错点**:WAN 的"广"不是指距离单一概念,而是跨地理区域。因特网(Internet)是 WAN 的典型代表。

#### 3.2 按传输技术分

- **广播式网络**:所有节点共享一条通信信道,一发全收(如以太网、WiFi)
- **点对点网络**:一对一通信,需要路由转发(如广域网)

#### 3.3 按拓扑结构分(可能考画图)

星型、总线型、环型、树型、网状型。

📌 **重要结论**:

- 总线型——一根线挂所有节点,**一断全断**,存在冲突问题
- 星型——中心节点一挂全挂,但单节点故障不影响其他
- 环型——令牌环网用,单向传输
- 网状型——可靠性最高,广域网常用

#### 3.4 按使用者分

公用网(如电信)、专用网(如军队、银行内部)。

#### 3.5 按传输介质分

有线网、无线网。

### 第四节 性能指标(**重灾区,必考**)

这一节几乎每年都有计算题,务必吃透。

#### 4.1 速率(Data Rate)

数据的传送速率,单位是 **bit/s**(也写作 b/s、bps)。

📌 **极易错**:

- 网络里的 K、M、G 是 **10 进制**:1 Kbps = 10³ bps,1 Mbps = 10⁶ bps
- 存储里的 K、M、G 是 **2 进制**:1 KB = 2¹⁰ B = 1024 B
- **b 是 bit(位),B 是 Byte(字节),1 B = 8 b**

#### 4.2 带宽(Bandwidth)

原指信号的频带宽度(Hz),在计算机网络中指**单位时间内从一点到另一点所能通过的最高数据率**,单位 bit/s。

你可以理解为"水管的粗细",它是**理论最大值**。

#### 4.3 吞吐量(Throughput)

单位时间内**实际**通过某个网络的数据量。吞吐量 ≤ 带宽。

#### 4.4 时延(Delay)—— **核心考点**

数据从网络一端传送到另一端所需时间,由四部分组成:

总时延=发送时延+传播时延+处理时延+排队时延总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延总时延=发送时延+传播时延+处理时延+排队时延

**① 发送时延(传输时延)**

发送时延=数据长度(bit)发送速率(bit/s)发送时延 = \frac{数据长度(bit)}{发送速率(bit/s)}发送时延=发送速率(bit/s)数据长度(bit)​

主机把数据推到线路上所花的时间。

**② 传播时延**

传播时延=信道长度(m)传播速率(m/s)传播时延 = \frac{信道长度(m)}{传播速率(m/s)}传播时延=传播速率(m/s)信道长度(m)​

信号在介质中跑的时间。电磁波在铜线中约 2×10⁸ m/s,光纤中约 2×10⁸ m/s,真空中 3×10⁸ m/s。

**③ 处理时延**:路由器分析包头、查路由表的时间 **④ 排队时延**:在路由器队列中等待的时间

📌 **必考辨析题**:

- 发送时延和数据长度有关,和距离无关
- 传播时延和距离有关,和数据长度无关
- **增大带宽只能减小发送时延,减小不了传播时延**(这是极经典的考点)

#### 4.5 时延带宽积

时延带宽积=传播时延×带宽时延带宽积 = 传播时延 × 带宽时延带宽积=传播时延×带宽

物理意义:**链路上正在传输的比特数**(把链路想象成一根水管,这是管子里水的体积)。

#### 4.6 往返时间 RTT(Round-Trip Time)

从发送方发出数据,到收到接收方确认,总共经历的时间。

RTT≈2×传播时延+其他RTT ≈ 2 × 传播时延 + 其他RTT≈2×传播时延+其他

#### 4.7 利用率

- **信道利用率**:某信道有百分之几的时间是被利用的
- **网络利用率**:全网信道利用率的加权平均

📌 **反直觉考点**:**利用率并非越高越好**。当利用率接近 1 时,时延急剧增大(排队论结论)。一般工程上控制在 50% 以下。

公式:$$D = \frac{D_0}{1-U}$$(DD D 是当前时延,D0D_0 D0​ 是空闲时延,UU U 是利用率)

### 第五节 分层体系结构(**核心重点**)

#### 5.1 为什么要分层

一句话考点答案:**将庞大复杂的问题转化为若干较小的局部问题,降低设计难度,便于实现、调试、维护和标准化**。

分层原则:

1. 各层功能明确、相互独立
2. 层间接口清晰,交互信息最少
3. 层数适中(太少不灵活,太多笨重)

#### 5.2 几个核心概念(名词解释高频)

**实体(Entity)**:任何可发送或接收信息的硬件或软件进程。

**对等实体(Peer Entity)**:不同机器上同一层的实体。

**协议(Protocol)**:**控制两个对等实体进行通信的规则的集合**。协议三要素:

- **语法**:数据与控制信息的结构或格式(长什么样)
- **语义**:需要发出何种控制信息、完成何种动作、做出何种响应(干什么)
- **同步**:事件实现顺序的详细说明(什么时候做)

📌 **必背三要素**,年年考。

**服务(Service)**:下层为相邻上层提供的功能调用。**协议是水平的,服务是垂直的**。

**SAP(服务访问点)**:相邻两层实体交互的地方。例:

- 传输层的 SAP 是**端口**
- 网络层的 SAP 是 **IP 地址**
- 数据链路层的 SAP 是 **MAC 地址**

**PDU(协议数据单元)**:对等层之间传送的数据单位。

- 应用层:**报文(Message)**
- 传输层:**报文段(Segment)** / 用户数据报(UDP)
- 网络层:**分组/数据报(Packet/Datagram)**
- 数据链路层:**帧(Frame)**
- 物理层:**比特(Bit)**

📌 **考试神坑**:每层的 PDU 名称必须记准。看到"报文段"就该反应到传输层,看到"帧"就该反应到数据链路层。

#### 5.3 OSI 七层模型

自下而上:**物 → 数 → 网 → 传 → 会 → 表 → 应**

记忆口诀:**物数网传会表应**,或者"**物联网传话表演**"。

|层|功能|典型协议/设备|
|---|---|---|
|应用层|为应用进程提供服务|HTTP、FTP、SMTP、DNS|
|表示层|数据格式转换、加密、压缩|JPEG、ASCII|
|会话层|建立、管理、终止会话|—|
|传输层|端到端可靠传输|TCP、UDP|
|网络层|路由选择、分组转发|IP、路由器|
|数据链路层|成帧、差错控制、流量控制|以太网、交换机、网桥|
|物理层|比特传输|中继器、集线器|

#### 5.4 TCP/IP 四层模型

自下而上:**网络接口层 → 网际层 → 传输层 → 应用层**

#### 5.5 五层模型(教学常用,也是考试标准)

**物理层 → 数据链路层 → 网络层 → 传输层 → 应用层**

考试答题时,除非题目特别指定 OSI,否则**默认用五层模型**。

#### 5.6 三个模型的对应关系(必考画图)

```
OSI七层        五层          TCP/IP四层
─────────     ─────        ──────────
应用层 ┐
表示层 ├──── 应用层 ─────── 应用层
会话层 ┘
传输层 ────── 传输层 ─────── 传输层
网络层 ────── 网络层 ─────── 网际层
数据链路层 ┐                ┐
          ├── 数据链路层 ──┤ 网络接口层
物理层    ┘  物理层        ┘
```

#### 5.7 数据封装与解封装过程(**必考简答题**)

当主机 A 向主机 B 发送数据时:

**在 A 端(自顶向下,逐层加头)**:

1. 应用层:产生数据报文 M
2. 传输层:加上 TCP/UDP 头 → 报文段
3. 网络层:加上 IP 头 → IP 数据报
4. 数据链路层:加头加尾 → 帧
5. 物理层:转成比特流发出

**在 B 端(自底向上,逐层拆头)**:反过来,每层拆掉本层的头,交给上层。

📌 **考点**:中间经过的路由器**只处理到网络层**(拆到 IP 头就够了,做路由决策后重新封装下发);经过的交换机**只处理到数据链路层**。

### 📝 本讲典型考题自测

**题 1(计算题)**:一条链路带宽为 100 Mbps,长度为 1000 km,信号传播速率为 2×10⁸ m/s。现发送一个 10⁶ bit 的文件,求发送时延、传播时延、总时延。

<details> <summary>参考答案</summary>

- 发送时延 = 10⁶ / (100×10⁶) = 0.01 s = 10 ms
- 传播时延 = 10⁶ / (2×10⁸) = 5×10⁻³ s = 5 ms
- 总时延(忽略处理与排队) = 15 ms

</details>

**题 2(辨析)**:"带宽越大,传播时延越小"——对吗?

<details> <summary>参考答案</summary>

错。带宽影响的是发送时延,传播时延只和链路长度与传播速率有关。

</details>

**题 3(简答)**:简述协议的三要素。

**题 4(选择)**:路由器工作在( )层。 A. 物理层 B. 数据链路层 C. 网络层 D. 传输层

<details> <summary>参考答案</summary>

C。集线器在物理层,交换机/网桥在数据链路层,路由器在网络层,网关在更高层。

</details>
