### 第一节 数据链路层概述

#### 1.1 三个基本概念(必区分)

- **链路(Link)**:从一个节点到相邻节点的一段**物理线路**(无源)
- **数据链路(Data Link)**:在链路上加上**协议**和**硬件/软件**,构成的逻辑通道
- **帧(Frame)**:数据链路层的 **PDU**

📌 一句话理解:**链路是物理的,数据链路是逻辑的**。

#### 1.2 数据链路层的三个基本功能(**必背**)

1. **封装成帧**(组帧)
2. **差错控制**(检错、纠错)
3. **流量控制**(让发送方不要发太快)

有些教材会加上"链路管理""透明传输""可靠传输"等。重点记住上面三条。

#### 1.3 两种信道

- **点对点信道**:一对一通信(如 PPP)
- **广播信道**:一对多通信(如以太网),需要解决"**谁先说话**"的问题(MAC 协议)

### 第二节 封装成帧与透明传输

#### 2.1 封装成帧

在网络层 IP 数据报前后加上**帧头**和**帧尾**,构成一个帧。

- 帧头帧尾的作用之一:**帧定界**(让接收方知道帧从哪开始、到哪结束)
- 每种链路层协议都规定了**帧的最大长度 MTU**(Maximum Transmission Unit)

#### 2.2 透明传输(**重要考点**)

**问题**:如果帧的数据部分恰好出现了和"帧定界符"一样的字节,接收方会误判帧结束。

**透明传输**的含义:**不管数据是什么样的比特组合,都能正确传输**(数据对协议是"透明"的)。

#### 2.3 实现透明传输的两种方法

**① 字符填充(面向字节)**

发送方:在数据中遇到与定界符相同的字节,前面加上**转义字符 ESC**(如 0x1B)。 接收方:遇到 ESC 就丢弃它,保留后面的字节。 若数据中本来就有 ESC,则在 ESC 前再加一个 ESC。

**② 比特填充(面向比特,如 HDLC)**

帧定界符:`01111110`(6 个连续 1)。 发送方:扫描数据,**每 5 个连续 1 后插入一个 0**。 接收方:扫描数据,**每 5 个连续 1 后删除一个 0**。

📌 **经典考题**:对比特串 `0110111111111100` 进行比特填充。 解:从左扫描,找连续 5 个 1 后插 0: `0110111110(插)1111(到这里又5个1插0)10(剩余)0` 结果:`011011111**0**11111**0**100`

### 第三节 差错控制(**核心考点**)

#### 3.1 差错的来源

- **随机错**(热噪声引起,孤立)
- **突发错**(脉冲噪声引起,连续多位)

#### 3.2 差错控制方法分类

- **检错码**:能发现错误,不能纠正(如奇偶校验、CRC)
- **纠错码**:能发现并纠正(如海明码)

#### 3.3 奇偶校验码

- **奇校验**:加一位使总 1 的个数为奇数
- **偶校验**:加一位使总 1 的个数为偶数

📌 **缺点**:**只能检测奇数个错误**,偶数个错误检测不出来。

#### 3.4 CRC 循环冗余校验(**必考计算**)

CRC 是数据链路层最重要的差错检测方法。

**步骤**:

1. 设要发送的数据 MM M 长 kk k 位,生成多项式 G(x)G(x) G(x) 长 r+1r+1 r+1 位
2. 在 MM M 后面加 rr r 个 0,得到 2rM2^r M 2rM
3. 用 2rM2^r M 2rM 除以 G(x)G(x) G(x)(**模 2 除法**,即按位异或,不借位)
4. 余数 RR R(长 rr r 位)就是 **CRC 校验码**(也叫**帧检验序列 FCS**)
5. 发送 M+RM + R M+R(共 k+rk+r k+r 位)

接收方:用收到的数据除以 G(x)G(x) G(x),**余数为 0 则无差错**,否则有差错。

📌 **必练例题**: 设 M=101001M = 101001 M=101001,G(x)=x3+x2+1G(x) = x^3 + x^2 + 1 G(x)=x3+x2+1(即 `1101`),求 CRC。

解:

- r=3r = 3 r=3,2rM=1010010002^r M = 101001000 2rM=101001000
- 用 `101001000` ÷ `1101`(模 2 除法):

```
       110001
     ┌──────────
1101 │101001000
       1101
       ────
        1110
        1101
        ────
          0110
             0
          ────
           1100
           1101
           ────
            0010   ← 这里实际过程要严格按位
```

(完整过程略,**最终余数 R = 001**)

发送内容:`101001 001`

📌 **CRC 重要结论**:

- CRC **只能检错,不能纠错**(虽然理论上可以,但实际工程上只用作检错)
- rr r 位 CRC 能检出**所有长度 ≤ r 的突发错**
- 模 2 运算 = 异或运算(**不进位、不借位**)

#### 3.5 海明码(纠错码,选学但常考概念)

- 能纠正 1 位错的最少校验位数公式:2r≥k+r+12^r \geq k + r + 1 2r≥k+r+1
- 例:数据位 k=4k=4 k=4,需要校验位 r=3r=3 r=3(23=8≥4+3+12^3 = 8 \geq 4+3+1 23=8≥4+3+1)

### 第四节 流量控制与可靠传输(**重灾区**)

数据链路层和传输层都有这个机制,但**思想完全相同**,这里学透了,后面 TCP 就轻松。

#### 4.1 三种基本协议

##### ① 停止-等待协议(Stop-and-Wait)

**机制**:发送方发一帧 → 等待 ACK → 收到 ACK 再发下一帧。

**问题与解决**:

- **数据帧丢失** → 设置**超时计时器**,超时重传
- **ACK 丢失** → 同样超时重传(接收方需识别重复帧)
- **如何识别重复帧** → 帧加**1 比特序号**(0/1 交替)

**信道利用率**:

U=TDTD+RTT+TAU = \frac{T_D}{T_D + RTT + T_A}U=TD​+RTT+TA​TD​​

其中 TDT_D TD​ 是数据帧发送时延,TAT_A TA​ 是 ACK 发送时延。

📌 **核心问题**:**长肥管道(带宽大、距离远)上利用率极低**,因为发送方大部分时间在等 ACK。

##### ② 后退 N 帧协议 GBN(Go-Back-N)

**机制**:发送方可以**连续发送 N 帧**而不必等 ACK,接收方按**累积确认**方式回复。

**关键特性**:

- 发送窗口大小 WT≤2n−1W_T \leq 2^n - 1 WT​≤2n−1(n 是序号位数)
- 接收窗口大小 WR=1W_R = 1 WR​=1(**只接收按序到达的帧**)
- 接收方收到失序帧 → **直接丢弃**,只发上一次正确帧的 ACK
- 发送方超时 → **从出错帧开始,后面所有帧全部重传**

📌 **缺点**:出错时重传量大,信道利用率不高。

##### ③ 选择重传协议 SR(Selective Repeat)

**机制**:接收方**单独缓存**失序帧,只重传**真正出错**的那一帧。

**关键特性**:

- 发送窗口 WT≤2n−1W_T \leq 2^{n-1} WT​≤2n−1
- 接收窗口 WR≤2n−1W_R \leq 2^{n-1} WR​≤2n−1
- 通常 WT=WRW_T = W_R WT​=WR​
- 接收方对每一帧**单独确认**(非累积)
- 接收方需要**缓存空间**

📌 **三个协议对比**(**必背表**):

||停等|GBN|SR|
|---|---|---|---|
|发送窗口 WTW_T WT​|1|≤2n−1\leq 2^n-1 ≤2n−1|≤2n−1\leq 2^{n-1} ≤2n−1|
|接收窗口 WRW_R WR​|1|1|≤2n−1\leq 2^{n-1} ≤2n−1|
|确认方式|单个|累积|单个|
|重传范围|1 帧|多帧|1 帧|
|信道利用率|低|中|高|
|实现复杂度|低|中|高|

📌 **窗口大小为什么有这些限制?** 若不限制,接收方可能将"重传的旧帧"误认为是"新一轮的新帧"——这是一个**经典考题**,务必能口述。

#### 4.2 滑动窗口

- 发送窗口:发送方可连续发送但未确认的帧的范围
- 接收窗口:接收方期望收到的帧的范围
- 收到 ACK,窗口向前滑动

### 第五节 介质访问控制(MAC,**核心考点**)

广播信道下,**多个节点共享一条信道**,需要解决"碰撞"问题。

#### 5.1 信道划分介质访问控制

把信道分成多个**子信道**,各节点用自己的子信道。包括 FDMA、TDMA、CDMA、WDMA。

📌 **特点**:**无冲突**,但信道利用率不一定高(用户少时浪费)。

#### 5.2 随机访问介质访问控制(**重点**)

各节点想发就发,可能冲突,冲突后处理。

##### ① ALOHA 协议

- **纯 ALOHA**:想发就发,冲突就重传(最大利用率仅 18%)
- **时隙 ALOHA**:只能在时隙开始时发(最大利用率 37%)

##### ② CSMA(载波侦听多路访问)

发送前**先听信道**:

- **1-坚持 CSMA**:听到忙就一直听,空闲立即发(冲突概率高)
- **非坚持 CSMA**:听到忙等随机时间再听
- **p-坚持 CSMA**:空闲时以概率 p 发送

##### ③ **CSMA/CD(带冲突检测,以太网核心,必考)**

**适用场景**:有线以太网

**核心思想:边发边听,检测到冲突立即停止**

**工作流程**:

1. **先听后发**(载波侦听)
2. **边发边听**(冲突检测)
3. 检测到冲突 → 立即停止发送 → 发送**强化干扰信号(jam 信号,32-48 bit)**
4. 等待**随机时间**(二进制指数退避)再尝试

**二进制指数退避算法**:

- 第 nn n 次冲突后,从 {0,1,2,...,2k−1}\{0, 1, 2, ..., 2^k-1\} {0,1,2,...,2k−1} 中随机取一个数 rr r,等待 r×2τr \times 2\tau r×2τ(τ\tau τ 为单程传播时延)
- k=min⁡(n,10)k = \min(n, 10) k=min(n,10)
- 16 次仍冲突 → 放弃发送,报告上层

**最小帧长(必考公式)**:

Lmin=2τ×CL_{min} = 2\tau \times CLmin​=2τ×C

其中 τ\tau τ 是单程传播时延,CC C 是数据率。

📌 **理由**:必须保证**发送过程中能检测到冲突**,即发送时间 ≥ 2 倍传播时延(端到端来回)。

📌 **以太网最小帧长 = 64 字节(512 bit)**(必背数字)

📌 **争用期(冲突窗口)= 2τ**,以太网为 51.2 μs。

##### ④ **CSMA/CA(带冲突避免)**

**适用场景**:**无线局域网(WiFi,IEEE 802.11)**

**为什么不用 CSMA/CD?**

- 无线信号衰减快,**发送时无法检测冲突**(自己的信号太强,听不到别人的)
- 存在**隐蔽站问题**(A 和 C 都能听到 B,但 A 和 C 互相听不到)

**核心思想:发送前协商,避免冲突**

**机制**:

1. 信道空闲后,等待 **DIFS** 时间
2. 发送数据
3. 接收方等待 **SIFS** 时间后返回 **ACK**
4. 还可使用 **RTS/CTS** 机制(可选):发送方先发 RTS(请求发送),接收方回 CTS(允许发送),其他站点收到 CTS 后保持沉默

📌 **CSMA/CD vs CSMA/CA**(**必考辨析**):

||CSMA/CD|CSMA/CA|
|---|---|---|
|应用|有线以太网|无线 WiFi|
|冲突处理|检测冲突后停止|提前避免冲突|
|是否有 ACK|无(由上层负责)|**有 ACK**|
|关键技术|二进制指数退避|RTS/CTS、IFS|

#### 5.3 轮询访问介质访问控制

- **轮询协议**:主节点轮流询问从节点
- **令牌传递协议**:令牌环网,谁拿到令牌谁发送

### 第六节 局域网与以太网

#### 6.1 局域网的特点

- 地理范围小、传输速率高、误码率低
- 拓扑:星型、总线型、环型、树型
- 由 IEEE 802 委员会制定标准

#### 6.2 IEEE 802 标准(**记几个重要的**)

|标准|内容|
|---|---|
|802.3|**以太网(CSMA/CD)**|
|802.11|**无线局域网 WiFi(CSMA/CA)**|
|802.15|蓝牙|
|802.1Q|VLAN|

#### 6.3 MAC 子层与 LLC 子层

IEEE 802 把数据链路层划分为:

- **LLC(逻辑链路控制)子层**:与硬件无关(已基本不用)
- **MAC(介质访问控制)子层**:与硬件有关,负责介质访问

#### 6.4 MAC 地址(物理地址、硬件地址)

- 长度:**48 位(6 字节)**
- 表示:`00-1A-2B-3C-4D-5E`(十六进制)
- **全球唯一**,固化在网卡 ROM 中
- 前 24 位:厂商代码(IEEE 分配),后 24 位:厂商自编

📌 **特殊地址**:

- 全 1(`FF-FF-FF-FF-FF-FF`):**广播地址**
- 第一个字节最低位为 1:多播地址

#### 6.5 以太网帧格式(**必考画图**)

```
┌────────┬────────┬────────┬──────┬────────┬─────┐
│前导码  │目的MAC │ 源MAC  │ 类型 │  数据  │ FCS │
│ 8字节  │ 6字节  │ 6字节  │2字节 │46-1500 │4字节│
└────────┴────────┴────────┴──────┴────────┴─────┘
```

📌 **关键数字**:

- 数据部分:**46~1500 字节**(最小 46 是为了凑齐最小帧 64 字节)
- 最小帧:**64 字节**;最大帧:**1518 字节**
- MTU = **1500 字节**(最大数据)

#### 6.6 以太网的演进

|类型|速率|
|---|---|
|经典以太网(10BASE-T)|10 Mbps|
|快速以太网|100 Mbps|
|千兆以太网|1 Gbps|
|万兆以太网|10 Gbps|

### 第七节 数据链路层设备

#### 7.1 网桥(Bridge)

- 工作层:**数据链路层**
- 功能:**根据 MAC 地址转发帧**,过滤帧
- 隔离冲突域,**不隔离广播域**
- 已基本被交换机取代

#### 7.2 以太网交换机(Switch)

- 本质:**多端口的网桥**
- 工作层:**数据链路层**
- 每个端口是**独立的冲突域**
- **所有端口属于同一个广播域**
- **全双工通信,独享带宽**

**交换机自学习**(**必考**):

1. 收到帧 → 记录"**源 MAC + 端口号**"到 MAC 地址表
2. 查表找目的 MAC 对应的端口
3. 找到 → 从该端口转发(**单播**)
4. 找不到 → **泛洪**(向除入口外的所有端口转发)

📌 **交换机三种转发方式**:

- **直通式**:收到目的地址(6 字节)就转发,延迟最小,但不检错
- **存储转发**:收完整帧再转发,可检错,延迟大(**最常用**)
- **无碎片**:收 64 字节后转发(过滤碎片帧)

#### 7.3 冲突域与广播域(**极易错考点**)

- **冲突域**:可能发生冲突的范围
- **广播域**:广播帧能到达的范围

|设备|隔离冲突域|隔离广播域|
|---|---|---|
|集线器|否|否|
|网桥/交换机|**是**|否|
|路由器|**是**|**是**|

📌 **结论**:

- 想隔离冲突域 → 用交换机
- 想隔离广播域 → 用路由器(或 VLAN)

#### 7.4 VLAN(虚拟局域网)

- **作用**:在交换机上**逻辑划分广播域**
- 同一 VLAN 内可通信,**不同 VLAN 之间需通过路由器**
- 标准:**IEEE 802.1Q**(在以太网帧中插入 4 字节 VLAN 标签)

### 第八节 PPP 协议(点对点协议)

广域网点对点链路上最常用的协议。

#### 8.1 PPP 帧格式

```
┌──────┬──────┬──────┬──────┬────────┬──────┬──────┐
│ 标志 │ 地址 │ 控制 │ 协议 │ 信息   │ FCS  │ 标志 │
│ 0x7E │ 0xFF │ 0x03 │2字节 │≤1500   │2/4字节│ 0x7E│
└──────┴──────┴──────┴──────┴────────┴──────┴──────┘
```

#### 8.2 PPP 特点

- **面向字节**(因此用**字符填充**)
- 但用于同步链路时使用**比特填充**
- **不提供可靠传输**(不用序号、不重传,因为现代链路质量好)
- **不使用滑动窗口**

📌 易错点:**PPP 没有可靠传输**,这是和 HDLC 的区别。
