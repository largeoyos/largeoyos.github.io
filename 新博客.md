typedef struct{
    BiTree data;
    struct BiQNode *next;

这里会报错

报错原因分析
在 C 语言中，编译器的解析顺序是从上到下的：

别名尚未定义：当你写到 struct BiQNode *next; 这一行时，编译器还不知道 BiQNode 是什么。因为 BiQNode 作为别名，要等到最后一行大括号结束时才会被定义。

标签不存在：你省略了 struct 后面的标签名。如果你写的是 typedef struct BiQNode { ... } BiQNode;，编译器会知道 struct BiQNode 是一个合法的类型。但在你的代码里，struct BiQNode 对编译器来说是一个全新的、从未见过的类型。

正确的写法（二选一）
如果你需要定义链表节点（即结构体内部需要指向相同类型的指针），你必须给结构体起一个“标签名”。

方案 A：最稳妥的教科书写法（推荐）
标签名和别名都叫 BiQNode。

C++
typedef struct BiQNode {
    BiTree data;
    struct BiQNode *next; // 这里必须用标签名 struct BiQNode
} BiQNode; // 这里定义别名 BiQNode
方案 B：先定义别名，再定义结构体
这种写法在一些高级工程中也会见到：

C++
typedef struct BiQNode BiQNode; // 先声明 BiQNode 是 struct BiQNode 的别名

struct BiQNode {
    BiTree data;
    BiQNode *next; // 此时别名已存在，可以直接用
};
